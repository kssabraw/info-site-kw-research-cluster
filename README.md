# Multi-Tenant Keyword Discovery & Clustering Pipeline

A repeatable pipeline that turns raw keyword discovery into approved topic
plans for content site generation. Supports multiple sites with per-site
configuration, designed to scale across a portfolio of niches.

For architecture and design decisions, see [PROJECT_BRIEF.md](PROJECT_BRIEF.md).
For Claude Code context, see [CLAUDE.md](CLAUDE.md).

## What This Does

Given a niche-specific configuration file (seeds, taxonomies, thresholds),
this pipeline:

1. Discovers keywords via DataForSEO API
2. Mines competitor SERPs for additional keywords
3. Classifies keywords by intent
4. Generates embeddings for clustering
5. Clusters keywords into topics using HDBSCAN
6. Refines clusters using SERP overlap
7. Exports clusters for human review
8. Produces approved topics ready for article generation

The output is a set of topic plans (primary keyword + supporting keywords +
subfolder assignment) that downstream content generation systems consume.

## Prerequisites

- Python 3.11 or higher
- [uv](https://github.com/astral-sh/uv) for dependency management
- Accounts and API access:
  - Supabase project (Postgres + pgvector)
  - DataForSEO API credentials
  - OpenAI API key (for embeddings)
  - Anthropic API key (for LLM classification)

## Setup

### 1. Clone and install

```bash
git clone 
cd 
uv sync
```

### 2. Configure environment

```bash
cp .env.example .env
# Edit .env with your API keys and Supabase credentials
```

### 3. Deploy database schema

In the Supabase SQL editor, run `schema/schema.sql`:

```bash
# Option A: Copy/paste into Supabase SQL editor

# Option B: Use psql if you have direct database access
psql "$DATABASE_URL" -f schema/schema.sql
```

Verify deployment:

```sql
SELECT table_name FROM information_schema.tables 
WHERE table_schema = 'public' 
ORDER BY table_name;
-- Should list 14 tables
```

### 4. Configure your first site

Create a YAML config in `config/sites/`:

```bash
cp config/sites/retatrutide.yaml config/sites/yoursite.yaml
# Edit with your seeds, niche terms, etc.
```

Register the site in the database:

```sql
INSERT INTO sites (slug, domain, display_name, niche_description, config) 
VALUES (
    'yoursite',
    'yoursite.com', 
    'Your Site Display Name',
    'Description of the niche',
    '{}'::jsonb
);
```

### 5. Verify setup

```bash
uv run python -m pipeline.run --verify-setup
```

This checks:
- All API credentials work
- Database connection works
- Schema is deployed
- At least one site is registered

## Running the Pipeline

### Run a single phase

```bash
uv run python -m pipeline.run --site yoursite --phase 01
```

### Run all phases sequentially

```bash
uv run python -m pipeline.run --site yoursite --phase all
```

### Run a range of phases

```bash
uv run python -m pipeline.run --site yoursite --phase 01-07
```

### Re-run a phase (idempotent)

```bash
uv run python -m pipeline.run --site yoursite --phase 10 --force
```

The `--force` flag overrides idempotency checks and re-runs even if
the phase has previously completed.

## Pipeline Phases

| Phase | Name | Purpose |
|-------|------|---------|
| 00 | Concept mapping | LLM generates tangential concepts for the niche |
| 01 | Seed expansion | DataForSEO keyword_ideas on primary + tangential seeds |
| 02 | SERP fetching | Top 10 organic results for highest-volume keywords |
| 03 | URL + domain frequency analysis | Auto-derive competitor list from SERPs |
| 04 | URL-level keyword mining | ranked_keywords on hub URLs, positions 1-20 |
| 05 | Domain-level keyword mining | keywords_for_site on top competitor domains |
| 06 | Relevance filtering | Must-match terms + semantic similarity threshold |
| 07 | Volume enrichment | keyword_overview for final volume/CPC validation |
| 08 | Intent classification | Haiku assigns each keyword to intent taxonomy |
| 09 | Embedding generation | OpenAI embeddings on enriched keyword text |
| 10 | HDBSCAN clustering | Cluster within each intent bucket separately |
| 11 | SERP overlap refinement | Split/merge clusters based on SERP overlap |
| 12 | Review export and import | Human-in-the-loop approval via Google Sheet |

See [docs/pipeline-phases.md](docs/pipeline-phases.md) for detailed specs
of each phase.

## Adding a New Site

To launch keyword discovery for a new niche:

1. **Create site config** at `config/sites/{newslug}.yaml`
   (use existing site config as template)

2. **Register in database**:
```sql
   INSERT INTO sites (slug, domain, display_name, niche_description, config) 
   VALUES (...);
```

3. **Run pipeline**:
```bash
   uv run python -m pipeline.run --site newslug --phase all
```

4. **Review clusters** in the Google Sheet generated by Phase 12

5. **Approve** by editing the sheet and re-running Phase 12 to import

No pipeline code changes required. Each site is fully isolated.

## Checking Pipeline Status

### View recent jobs

```bash
uv run python -m pipeline.run --site yoursite --status
```

### Query pipeline_jobs directly

```sql
SELECT phase, status, started_at, completed_at, 
       output_summary->>'keywords_discovered' as keywords
FROM pipeline_jobs 
WHERE site_id = (SELECT id FROM sites WHERE slug = 'yoursite')
ORDER BY started_at DESC
LIMIT 20;
```

### View site stats

```sql
SELECT 
    (SELECT COUNT(*) FROM raw_keywords WHERE site_id = s.id) as keywords,
    (SELECT COUNT(*) FROM clusters WHERE site_id = s.id) as clusters,
    (SELECT COUNT(*) FROM topics WHERE site_id = s.id) as topics
FROM sites s
WHERE s.slug = 'yoursite';
```

## Troubleshooting

### "DataForSEO rate limit exceeded"

DataForSEO has request rate limits per second. The pipeline includes
automatic retry with backoff, but heavy usage may still hit limits.

Solutions:
- Wait a few minutes and re-run with `--force`
- Reduce `max_keywords_for_serp_fetch` in site config
- Check DataForSEO dashboard for rate limit settings

### "OpenAI API key invalid"

Verify the key in `.env` starts with `sk-` and is from
[platform.openai.com](https://platform.openai.com/api-keys).

### "pgvector extension not available"

Supabase has pgvector enabled by default on Pro tier. If on Free tier,
verify in Database → Extensions that `vector` is enabled.

### "Phase X failed midway through"

The pipeline is idempotent — re-run with `--force` to retry. Job
tracking ensures partial progress isn't duplicated.

### "Cluster count is very different from target"

The clustering algorithm doesn't directly hit cluster count targets.
If results are way off (e.g., 50 clusters instead of 500):
- Check `min_cluster_size` in config (lower = more clusters)
- Check the keyword pool size (more keywords usually = more clusters)
- Review intent distribution — heavily skewed intents produce
  imbalanced cluster counts
